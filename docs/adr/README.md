# ADR Index

Architecture Decision Records capture durable choices that shape this repository and product.

## How To Use ADRs

- Create a new ADR when a decision changes future architecture, product direction, data modeling, or ways of working.
- Keep ADRs short and concrete.
- Prefer updating status over silently replacing old decisions.

## Status Meanings

- Proposed
- Accepted
- Superseded
- Deprecated
