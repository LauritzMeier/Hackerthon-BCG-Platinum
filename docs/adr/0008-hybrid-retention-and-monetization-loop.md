# ADR-0008: Hybrid Retention And Monetization Loop

- Status: Accepted
- Date: 2026-04-09

## Context

The product must solve two things at once:

- create a reason for regular weekly use
- create believable preventive revenue

If we optimize only for diagnostics conversion, the app risks becoming episodic.
If we optimize only for coaching engagement, the commercial story weakens.

## Decision

Use a hybrid product loop:

- coaching-led weekly engagement creates recurring value
- compass-driven diagnostics or support offers create monetization when justified

The Longevity Compass remains the source of truth for both.

This means:

- the weekly plan and coach loop are always present
- offers appear only when the pillar model makes them relevant and explainable
- the app should support both recurring engagement and episodic conversion in one coherent experience

## Consequences

- the first MVP should include both a weekly coaching loop and a next-best-offer surface
- monetization should feel earned because it is traceable back to the compass
- product measurement should track both engagement and conversion, not only one of them

## Alternatives Considered

- diagnostics-led app with minimal retention design
- coaching-led app with no strong commercial path
