# Recovery-First New User Plan

## Goal

Make the first-time experience feel trustworthy, simple, and relevant for a user who is trying to recover from a serious health scare and start a longevity journey without being overwhelmed.

## Product Principles

1. Show what the app already knows before asking the user to do more.
2. Make the coach sound informed, not generic.
3. Recommend one primary support option that clearly fits the current situation.
4. Explain what data is still missing, especially around meals and symptom tracking.
5. Keep every recommendation connected to a clear user benefit.

## Phase 1: Orientation

Ship a start-here layer on the main page:

- what we already know
- what we still need
- what to do first

The user should understand in less than 30 seconds:

- what the app sees from the smartwatch
- what medical context is already available
- why nutrition is still only broadly personalized

## Phase 2: Recovery-Aware Coach

The coach should:

- reference the last doctor context on file
- acknowledge connected wearable data
- explain when advice is broad because meal data is missing
- respond safely when the user mentions a heart event or recent scare

## Phase 3: Better Support Offers

The offers surface should:

- show one best next support option
- keep secondary options short and relevant
- avoid persona-demo bundles as the primary story
- support drill-down detail for why now, what is included, what data was used, and what is still missing

Priority support types:

1. clinical review when risk clarity is needed
2. guided recovery movement when consistency is low
3. nutrition reset with meal tracking when personalization is weak
4. appointment prep when the user needs help turning care context into action

## Phase 4: Follow-Through

After a user accepts support, the app should:

- turn it into a weekly next step
- update the coach prompts
- track whether the user completed the setup behavior
- refresh the recommendation once new data arrives

## Success Signals

- user can explain why the primary offer was recommended
- user opens drill-down details before booking or saving
- user starts meal tracking when prompted
- user returns to the coach with better context instead of dropping out
